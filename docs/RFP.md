# RFP: DeepGuard — AI生成動画検出SaaS

## 1. プロジェクト概要

### 1.1 背景
2025-2026年、Sora 2/Veo 3等のAI動画生成ツールの進化により、AI生成動画と実写の区別が困難に。
特に**戦争・紛争映像のフェイク**がSNSで拡散され、世論操作・パニック誘発のリスクが深刻化。

### 1.2 ミッション
「動画がAI生成か人間制作かを判定するAPI/SaaSを提供し、偽情報の拡散を防止する」

### 1.3 プロダクト名
**DeepGuard** (ディープガード)

## 4. 機能要件

### 4.1 MVP（Phase 1）
1. **動画アップロード解析**
   - 動画ファイル（MP4/MOV/WebM）をアップロード → AI/人間判定
   - 判定結果: スコア(0-100) + 判定(AI生成/人間制作/不明) + 根拠
   
2. **URL解析**
   - YouTube/Twitter/TikTok URLを入力 → 動画取得 → 解析

3. **解析エンジン（マルチシグナル）**
   - **フレーム解析**: CNN/ViTで各フレームのAI生成確率を推定
   - **時系列解析**: フレーム間の時間的一貫性チェック（LSTM/Transformer）
   - **音声解析**: 音声の合成度チェック（環境音の不自然さ）
   - **メタデータ解析**: EXIF/codec/圧縮アーティファクト分析
   - **物理整合性**: 光源方向、影の整合性、物体の運動法則チェック

4. **戦争映像特化チェック**
   - 爆発のパーティクル物理シミュレーション整合性
   - 煙・炎の流体力学的リアリティ
   - 兵器・軍用車両の3Dモデル一致度
   - 地理的整合性（背景の建築物・地形とGeoINT照合）
   - 音響解析（爆発音の反響パターンが環境と一致するか）

5. **REST API**
   ```
   POST /api/v1/analyze
   POST /api/v1/analyze-url
   GET  /api/v1/results/{id}
   ```

6. **Webダッシュボード**
   - ドラッグ&ドロップアップロード
   - 解析結果の可視化（ヒートマップ、タイムライン）
   - 履歴管理

### 4.2 Phase 2（将来）

## 5. AI生成動画の見分け方（技術的根拠）

### 5.1 フレームレベルの手がかり
| チェック項目 | AI生成の特徴 | 検出方法 |
|---|---|---|
| 手指 | 6本指、関節の不自然さ | CNN + 骨格検出 |
| 顔 | 左右非対称、瞬きパターン異常 | 顔ランドマーク + 時系列分析 |
| テキスト | 文字が溶ける、読めない | OCR + 一貫性チェック |
| 反射・影 | 光源方向の不一致 | 物理ベースレンダリング逆算 |
| エッジ | フリッカリング、境界ぼけ | 高周波解析 |
| テクスチャ | 過度に滑らか or 繰り返しパターン | フーリエ解析 |

### 5.2 時間軸の手がかり
| チェック項目 | AI生成の特徴 | 検出方法 |
|---|---|---|
| オプティカルフロー | 物体の動きが物理法則に反する | フロー推定 + 物理シミュレーション |
| 時間的一貫性 | フレーム間で物体が消失/出現 | オブジェクトトラッキング |
| 圧縮アーティファクト | AI特有のノイズパターン | FFT + 周波数領域解析 |

### 5.3 戦争映像特有の手がかり
| チェック項目 | フェイクの特徴 | 検出方法 |
|---|---|---|
| 爆発 | パーティクルが均一すぎる | 流体力学シミュレーション比較 |
| 煙 | 不自然な拡散パターン | ボリューメトリック解析 |
| 破壊 | 建物の崩壊が物理的に不自然 | 剛体力学シミュレーション |
| 兵器 | モデルと実物の不一致 | 画像検索 + 3Dモデル照合 |
| 地形 | 衛星画像と不一致 | GeoINT + 衛星画像API |
| 音響 | 爆発音と距離の不整合 | 音速計算 + 反響解析 |

## 6. 技術アーキテクチャ

### 6.1 スタック
- **Frontend:** Next.js 15 + Tailwind CSS
- **Backend:** Python FastAPI
- **ML Pipeline:** PyTorch + HuggingFace Transformers
- **DB:** PostgreSQL + Redis（キャッシュ/キュー）
- **Storage:** S3互換（動画一時保存）
- **Queue:** Celery + Redis（非同期解析ジョブ）
- **Deploy:** Docker → Railway or Fly.io（MVP）

### 6.2 MLモデル構成
```
入力動画
  ├── フレーム抽出 (FFmpeg)
  │     ├── ViT/EfficientNet (空間解析)
  │     ├── 顔検出 + ランドマーク (MediaPipe)
  │     └── テキスト検出 (EasyOCR)
  ├── 時系列解析 (Video Transformer)
  │     ├── オプティカルフロー (RAFT)
  │     └── 時間的一貫性スコア
  ├── 音声解析 (Whisper + カスタムモデル)
  │     ├── 音声合成度
  │     └── 環境音整合性
  └── メタデータ解析
        ├── コーデック/圧縮情報
        └── EXIF/生成ツール痕跡
          ↓
    アンサンブルスコアリング
          ↓
    判定: AI生成確率 0-100%
```
